# code_2026_04_10: paper-order replication bundle with Overleaf sync

This folder contains the active trimmed replication package for the current manuscript version stored in:

- `/Users/marcolemoglie_1_2/Library/CloudStorage/Dropbox/Applicazioni/Overleaf/Educated to Be Mothers-slides/AA_paper_new.tex`

The active workflow is now:

1. run the package locally and write every generated table/figure to this folder's internal `results/` tree;
2. once the run succeeds, sync the manuscript-facing outputs to the Overleaf project.

This avoids permission failures when the code tries to write directly into Dropbox/Overleaf during the Stata run, while still ensuring that the final manuscript uses the updated outputs.

The package is designed to expose one clean replication workflow with:

1. final labeled analysis datasets;
2. a single paper-order Stata file;
3. the external-source scripts needed to document the retained appendix inputs;
4. only the figure inputs and outputs that are actually needed to reproduce the current paper.

- `results/main_text/`
  - tables and figures used in the main text of the current manuscript
- `results/appendix/`
  - tables and figures used in the appendix of the current manuscript
- `logs/`
  - logs generated by package scripts

## How to regenerate external-source inputs

The package includes the scripts used upstream to download and clean external sources:

- `code/external_sources/scripts/download_official_sources.sh`
- `code/external_sources/scripts/download_ine1950_activity_docs.sh`
- `code/external_sources/scripts/clean_external_sources.py`
- `code/external_sources/scripts/build_pre73_crisis_proxies.py`
- `code/external_sources/stata_active/external_revision_02_crisis73_macro_proxies_wholeprovince.do`
- `code/external_sources/stata_active/revision_08_birthplace_crisis73.do`

The logic is:

1. download official raw material where scripted downloads exist;
2. clean or parse those files, preserving the whole-province 1950 proxy values used for the oil-shock block;
3. run the active external revision do-files to produce the final external `.dta` inputs;
4. run `code/prep_data/01_build_final_analysis_data.do` to copy the final external inputs into the package.

The capital/rest-of-province oil-shock specifications are retained only in `code/external_sources/deprecated_capital_rest/` because the OCR at that level is too fragile for the current revision design.

For the `1973` robustness block, the current review path uses the whole-province proxy. The capital/rest-of-province variant is archived but not selected.

## Notes on raw data size and package design

This package is “complete” in the sense relevant for replication within this project workspace:

- it contains the final minimal `.dta` analysis files actually needed by the current paper;
- it contains the original and revised code used to generate the tables and figures;
- it contains the external-source scripts and documentation needed to trace the externally-built inputs back to official sources.

The very large raw internal datasets are **not duplicated** inside this folder because they already live in stable project paths under `PURGE/original_data/` and `PURGE/output_data/`. Duplicating them would create a much larger package without adding clarity. Instead, the package documents exactly which raw files are used and provides the scripts that turn them into the final labeled `.dta` files inside `analysis_data/`.

For the current manuscript objects, the package is self-contained at the final-analysis stage:

- the final `.dta` files required by the paper are stored in `analysis_data/`;
- the final figure inputs needed by the paper-order script are stored in `figure_inputs/`;
- rerunning `code/Main_analysis_paper_order.do` from a clean `results/` folder should therefore reproduce the package outputs without re-reading the larger project raw-data directories.
